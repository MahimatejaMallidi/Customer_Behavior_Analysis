# Customer_Behavior_Analysis
data analytics project showcasing customer behavior analysis using python, sql, power Bi

🛍️ Customer Shopping Behavior Analysis
A full-stack data analytics project analyzing 3,900 customer records across purchasing patterns, demographics, product preferences, and subscription behavior — using Python, MySQL, and Power BI.

📌 Project Overview
This project explores a retail customer dataset to uncover actionable business insights around revenue drivers, discount effectiveness, customer loyalty, and product performance. The analysis pipeline spans data cleaning and transformation in Python, structured querying in MySQL, and interactive visualization in Power BI.

🗂️ Dataset

Source: Customer Shopping Behavior Dataset
Records: 3,900 customers
Features: 18 columns including Age, Gender, Category, Purchase Amount, Review Rating, Subscription Status, Shipping Type, Discount Applied, Previous Purchases, Payment Method, Frequency of Purchases

Product Categories: Clothing · Footwear · Outerwear · Accessories

🛠️ Tools & Technologies
ToolPurposePython (pandas)Data cleaning, feature engineering, EDAMySQLStructured querying & business analysisPower BIInteractive dashboard & data visualization

🐍 Python — Data Cleaning & Feature Engineering
Key steps performed:

Handled missing values in Review Rating using category-wise median imputation
Standardized column names (lowercase, underscores)
Engineered age_group feature using quartile-based binning → Young Adult, Adult, Middle-aged, Senior
Mapped frequency_of_purchases text values to numeric purchase_frequency_days
Identified and dropped the redundant promo_code_used column (perfectly correlated with discount_applied)
Exported cleaned DataFrame to MySQL using SQLAlchemy + PyMySQL

Libraries: pandas, sqlalchemy, mysql-connector-python

🗄️ MySQL — Business Analysis Queries
Ten analytical queries were written to answer key business questions:
#QuestionQ1Total revenue generated by male vs. female customers
Q2Customers who used a discount but still spent above average
Q3Top 5 products by average review rating
Q4Average purchase amount: Standard vs. Express shipping
Q5Do subscribed customers spend more? (avg spend + total revenue)
Q6Top 5 products with the highest discount application rate
Q7Customer segmentation: New / Returning / Loyal (by previous purchases)
Q8Top 3 most purchased products within each category (Window Function)
Q9Are repeat buyers (>5 purchases) more likely to subscribe?
Q10Revenue contribution by age group
SQL concepts used: Aggregate functions, Subqueries, CTEs (WITH), CASE WHEN, ROW_NUMBER() window function, GROUP BY, ORDER BY

📊 Power BI — Dashboard
An interactive Power BI dashboard was built to visualize:

Revenue split by gender
Customer segmentation distribution (New / Returning / Loyal)
Top products by revenue and rating
Subscription vs. non-subscription spend comparison
Age group revenue contribution
Discount usage patterns across product categories


💡 Key Insights

Subscribed customers generate higher average spend and total revenue than non-subscribers
Loyal customers (>10 previous purchases) represent a disproportionately high share of total revenue
Certain products maintain high review ratings despite frequent discounting, suggesting strong brand equity
Express shipping customers tend to have slightly different average spend compared to Standard shipping users
Middle-aged and Adult age groups contribute the most to overall revenue

🚀 How to Run

Python Notebook
Open Customer_Behavior_Python.ipynb in Jupyter. Ensure pandas and sqlalchemy are installed.

bash   pip install pandas sqlalchemy pymysql mysql-connector-python

MySQL
Create a database named Customer_Behavior, then run Customer_Behavior_Mysql.sql.
The Python notebook pushes the cleaned data to the mytable table automatically.
Power BI
Open Customer_Behavior_Dashboard.pbix in Power BI Desktop.
Connect to your local MySQL database if refresh is needed.
